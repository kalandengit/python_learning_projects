# Beta Testing Program

## Ideal beta users
- SaaS companies with weekly release notes or support articles.
- Agencies creating client SOPs.
- Product managers maintaining onboarding guides.
- QA teams documenting workflows.
- Customer success teams writing help center content.

## Beta interview script
1. What documentation do you create every week?
2. Which tools do you use today?
3. How long does one document take from start to publish?
4. What breaks or becomes outdated most often?
5. Show us one workflow you would want automated.
6. Upload screenshots and generate a guide.
7. Rate the first draft from 1-10.
8. What would make this worth paying for?

## Feedback taxonomy
- Activation blocker
- AI quality issue
- Editor issue
- Export issue
- Collaboration request
- Integration request
- Billing issue
- Security/compliance concern

## Beta success threshold
The product is ready for broader launch when at least 5 beta teams say they would be disappointed if they could no longer use it.
