# Section 11 — UX Architecture

Frontend: React 19 + TypeScript 5 SPA (Vite), MUI v7, TanStack Query v5, React Router 7
(ADR-016). Scaffold in `frontend/`.

## 11.1 Information architecture (site map)

```
AMS Web
├── Home (role-adaptive dashboard)
├── Visits
│   ├── My visits (host) / New visit / Visit detail
│   ├── Today at site (reception) / Walk-in registration
│   └── Group visits
├── Badges
│   ├── Badge search / Badge detail (timeline = event stream)
│   ├── Issue badge / Replace badge / Badge stock (per site)
│   └── My badge (self-service: report lost)
├── Access
│   ├── Request access / My requests
│   ├── Approvals inbox / Request detail / Delegations
│   └── Policies (versioned) / Grants explorer
├── Security Operations
│   ├── SOC dashboard (occupancy, alarms, validity)
│   ├── Live decisions feed / Alarms / Watchlist
│   └── Evacuation console + Muster (mobile-first)
├── Compliance
│   ├── Audit search / Evidence queries / Report packs
│   └── Access-review campaigns
├── Administration
│   ├── Sites & zones / Devices / Config snapshots
│   ├── Cardholders / Bulk import
│   └── Webhooks / Notification templates
└── Kiosk (separate locked-down bundle: welcome → scan → docs → badge → goodbye)
```

**Navigation model:** persistent left rail (collapses to bottom bar < 900 px); global
site-switcher in the app bar scopes every view; breadcrumbs on detail pages; kiosk and
warden-muster are separate route bundles with kiosk-mode chrome (no rail). All navigation
keyboard-reachable (WCAG 2.2).

## 11.3 Security Operations dashboard spec

- **Layout:** 12-col grid; top row KPI tiles (on-site count, visitors on-site, active
  alarms, open escalations, edge gateways offline); centre: site/zone occupancy treemap
  with capacity colour thresholds; right rail: live alarm feed (ack/assign); bottom:
  decisions-per-minute sparkline + deny-reason breakdown.
- **Data:** occupancy via SSE (≤ 5 s lag, NFR tied to FR-043); alarms via Redis pub/sub
  fan-out; validity KPI from `credential_validation_snapshot` freshness metric.
- **Degradation:** every live tile shows a data-age chip ("live" / "12 s ago" / "stale");
  a stale feed switches the tile to amber with last-known values — never blank panels
  during incidents (same philosophy as ADR-020).
- **Actions:** alarm acknowledge (audit-logged), badge quick-revoke (step-up FIDO2
  re-auth), evacuation activation entry point (double-confirm + reason).

## 11.4 Design system

- **Tokens (MUI v7 theme):** brand palette with light/dark schemes via CSS variables
  (`theme.cssVariables = true`); semantic tokens `severity.permit` (green 700 on light) /
  `severity.deny` (red 700) / `severity.alarm` (orange 800) checked to ≥ 4.5:1 contrast on
  their surfaces; spacing scale 4 px; radius 8 px; type ramp: Inter, 14 px body, 1.5 line
  height; density presets (comfortable / compact for SOC).
- **Component strategy:** MUI v7 primitives wrapped once in `frontend/src/components/`
  (e.g., `AppDataGrid`, `StateChip`, `ConfirmDialog`) so design changes are single-point;
  no raw MUI imports in feature code (lint rule). Storybook documents each wrapper with
  axe-core checks per story.
- **Status is never colour-only:** every state chip pairs colour with icon + label
  (WCAG 1.4.1).

## 11.5 WCAG 2.2 AA plan (each item ↦ testable check)

| Requirement | Implementation | Automated check |
|---|---|---|
| Semantic structure | Landmarks (`header/nav/main`), one `h1`/page, lists for collections | axe-core `region`, `page-has-heading-one`, `heading-order` |
| Keyboard | All actions reachable; no traps; kiosk supports switch access; skip-link first | Playwright keyboard-walk e2e + axe `focusable-content` |
| Focus visible & not obscured | 2 px outline token, sticky headers padded so focus is never hidden (2.2 focus-not-obscured) | Playwright screenshot assert on focused elements |
| Focus order | DOM order = visual order; dialogs trap + restore focus | axe `tabindex`, RTL unit tests on dialogs |
| Contrast | Tokens validated ≥ 4.5:1 text, ≥ 3:1 UI components in both themes | axe `color-contrast` on every Storybook story (CI gate, zero critical) |
| Target size | Interactive targets ≥ 24×24 px (2.2 target-size); kiosk ≥ 48 px | Storybook interaction test measuring hit areas |
| Forms & errors | Labels always visible; errors linked via `aria-describedby`; no timeout without warning | axe `label`, RTL tests for error announcement |
| Screen reader | Live regions for occupancy/alarms (`aria-live=polite`, alarms `assertive`); decision feed announces batched summaries not every row | NVDA/VoiceOver scripted smoke per release (manual, checklist) |
| Motion & flashing | Reduced-motion media query honoured; no content flashes > 3/s | lint rule + review checklist |
| Language & reading | `lang` attribute, plain-language kiosk copy (visitor-facing at ~B1 level) | content review checklist |

CI gate: `npm run test:a11y` runs axe-core against every Storybook story and the five
critical user flows in Playwright; **zero critical violations** blocks merge (NFR-021).
